Architecture

MAARS runs an end-to-end research loop as three Multi-Agent stages glued by a file-based session DB. Python runtime owns control flow; agents do open-ended work.

01Goal & shape

The goal is end-to-end: from a raw idea to a finished paper, with every step persisted and recoverable. The shape is uniform: three stages, each Multi-Agent. Agents emit artifacts to disk; runtime orchestrates; the next agent reconstructs context from the persisted files. Complexity differs per stage, but the core is the same — Python owns the flow, agents execute open-ended work, state lives on disk.

02Runtime vs Agent

A hard line between what Python does deterministically and what an LLM does open-endedly.

Runtime · Python

Control flow & state

if / for / while, scheduling, retries, termination, checkpoint & resume, rate limiting, session DB I/O, SSE broadcast.

Agent · LLM

Open-ended work

Literature search, proposal drafting, code authoring & execution, review & critique, synthesis into prose.

04Stage inheritance

One base class for lifecycle and SSE; two branches for the two fundamentally different execution shapes.

Stage # lifecycle + SSE (_send) + LLM streaming (_stream_llm) ├── ResearchStage # Multi-Agent (decompose · parallel execute · evaluate loop) └── TeamStage # IterationState + primary / reviewer loop ├── RefineStage └── WriteStage # overrides _execute(): loop then polish + metadata sub-steps

05End-to-end flow

Three stages connected only by files on disk. Each arrow is a filesystem artifact, not an in-memory message.

stage 01

Refine

Explorer ↔ Critic

Explorer drafts a proposal from the raw idea. Critic reviews, produces {issues, resolved}. They iterate through an IterationState until issues is empty.

in → idea.md
out → refined_idea.md
→
stage 02 · engine

Research

Calibrate · Strategy · Decompose · Execute ↔ Verify · Evaluate

The core engine. Calibrate defines atomicity; Strategy proposes approach; Decompose produces a DAG of atomic tasks; each task runs in a Docker sandbox, verified, then the whole batch is evaluated — looping with a strategy_update when critical gaps exist.

in → refined_idea.md
out → artifacts/ · tasks/ · evaluations/
→
stage 03

Write

Writer ↔ Reviewer → Polish · Metadata

Writer reads the research artifacts and drafts the paper; Reviewer critiques; they iterate. After the loop converges, a Polish sub-step does a single-pass LLM refinement and a deterministic Metadata appendix is attached.

in → artifacts/
out → paper.md → paper_polished.md

Stage responsibilities

Stage Pattern Role
Refine Multi-Agent (Explorer + Critic) Intent → actionable research goal
Research Multi-Agent (Decompose + Execute + Evaluate) Decompose → Execute ↔ Verify → Evaluate
Write Multi-Agent (Writer + Reviewer) + polish / metadata Synthesize artifacts into a polished paper

06SSE event model

The frontend gets streaming notifications, but canonical state always comes from the session DB.

Conventions

  • Unified event shape: {stage, phase?, chunk?, status?, task_id?, error?}
  • With chunk — in progress, streaming text, left panel renders token-by-token.
  • Without chunk — done signal; DB is already written, right panel refetches from DB.
  • With status — intermediate task state (running / verifying / retrying).
  • DB is the single source of truth — SSE is notification only.

Broadcast skeleton

def _send(self, chunk=None, **extra):
    event = {"stage": self.name}
    if self._current_phase:
        event["phase"] = self._current_phase
    if chunk:
        event["chunk"] = chunk
        self.db.append_log(...)
    event.update(extra)
    self._broadcast(event)

08Session storage

One run, one directory. Every agent output is a file — grep-friendly, diffable, reproducible.

results/{session}/ ├── idea.md # user input ├── proposals/ critiques/ # Refine: rounds ├── refined_idea.md # Refine: final ├── calibration.md # Research: calibration ├── strategy/ plan_tree.json # Research: strategy · plan ├── plan_list.json # Research: flat task list ├── tasks/ artifacts/ # code · figures · data ├── evaluations/ # Research: eval rounds ├── results_summary.{json,md} # Research: summary ├── drafts/ reviews/ # Write: rounds ├── paper.md # Write: draft ├── paper_polished.md # Write: polished (sub-step) ├── meta.json # metadata (tokens, score) ├── log.jsonl # SSE stream log ├── execution_log.jsonl # Docker exec log └── reproduce/ # Dockerfile · run.sh

09Code layout
