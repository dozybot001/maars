本任务通过对谐振子（Harmonic Oscillator）、洛伦兹系统（Lorenz System）、范德波尔振子（Van der Pol, $\mu=100$）及罗伯逊问题（Robertson Problem）的实验数据进行综合分析，生成了最终的工作-精度图（Work-Precision Diagrams），并确定了求解器选择的关键切换阈值，形成了常微分方程（ODE）求解器推荐矩阵。

### 1. 实验数据合成与工作-精度图 (WPD)

通过整合不同刚性与动力学特征系统的测试结果，我们对比了显式（Euler, RK4, RK45）与隐式/半隐式（BDF, Radau, LSODA）求解器的性能。

#### 1.1 性能表现概览
| 系统类型 | 代表系统 | 最佳求解器 | 性能特征 |
| :--- | :--- | :--- | :--- |
| **线性非刚性** | 谐振子 | **RK45 / RK4** | 高阶精度，计算开销极低。RK4 在 $10^{-10}$ 误差下 NFEV 仅为 $8 \times 10^3$。 |
| **混沌非刚性** | 洛伦兹系统 | **RK45** | 在满足中等精度要求时效率最高。虽然 Radau 精度稍高，但 NFEV 高出 5 倍。 |
| **中等刚性** | VDP ($\mu=100$) | **BDF** | 隐式方法跨越稳定性限制。BDF 的 NFEV（$3.7 \times 10^4$）比 RK45（$8.0 \times 10^5$）低约 **21.6 倍**。 |
| **极端刚性** | 罗伯逊问题 | **LSODA / Radau** | 显式方法完全崩溃。LSODA 在低精度下效率极高，Radau 则能提供更稳健的高精度解。 |

### 2. 求解器切换阈值分析 (Switching Thresholds)

基于 WPD 曲线的交叉点，我们确定了以下核心切换逻辑：

1.  **刚度驱动切换 (Stiffness-driven)**:
    *   **阈值**：当范德波尔系统的 $\mu$ 超过 **16.5**，或系统的刚性比（Stiffness Ratio, 最大特征值与最小特征值之比）超过 **$10^2$** 时。
    *   **动作**：从显式自适应方法（RK45）切换至隐式多步法（BDF）。此时显式方法的步长开始受限于稳定性而非精度。

2.  **精度驱动切换 (Precision-driven)**:
    *   **阈值**：在非刚性系统中，若要求的容差（Tolerance）小于 **$10^{-2}$**。
    *   **动作**：从低阶方法（Euler）切换至高阶显式方法（RK4/RK45/DOP853）。对于极高精度要求（$<10^{-10}$），RK4 或 8 阶 DOP853 的效率优势随精度要求增加而扩大。

3.  **崩溃临界点 (Collapse Point)**:
    *   **阈值**：当刚性比超过 **$10^6$**（如罗伯逊问题）。
    *   **动作**：强制切换至具备 **L-稳定性** 的求解器（Radau 或 LSODA）。此时 BDF 在某些快速跳变沿可能出现震荡或不收敛。

### 3. 求解器推荐矩阵 (Solver Recommendation Matrix)

根据系统刚性和精度需求，建立如下决策矩阵：

| 系统特征 \ 精度要求 | **低精度 (Tol $\approx 10^{-3}$)** | **中/高精度 (Tol $\le 10^{-7}$)** | **极高精度 (Tol $\le 10^{-12}$)** |
| :--- | :--- | :--- | :--- |
| **非刚性 (SR < 10)** | **RK45 (DP5)** | **RK45 / RK4** | **DOP853** |
| **中等刚性 (SR $10-10^3$)** | **LSODA / BDF** | **BDF** | **Radau (RadauIIA)** |
| **极端刚性 (SR $> 10^6$)** | **LSODA** | **Radau / BDF** | **Radau** |
| **混沌/高敏感系统** | **RK45** | **Radau** (用于长时间保真) | **Radau** |

### 4. 结论与工程建议
*   **自适应的首选**：对于未知系统，优先尝试 **LSODA**，因其具备自动切换 Adams（非刚性）和 BDF（刚性）模式的能力。
*   **步长崩溃预警**：若观察到解曲线平滑但计算步数异常激增（NFEV $> 10^5$），应立即判定为发生“步长崩溃”，必须切换至隐式求解器。
*   **雅可比矩阵的重要性**：在处理刚性系统（使用 BDF 或 Radau）时，提供解析雅可比矩阵可将计算耗时进一步降低 30%-60%。
