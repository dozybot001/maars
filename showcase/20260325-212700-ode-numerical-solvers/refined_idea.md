# 研究计划书：常微分方程数值求解器的性能基准研究——跨刚性特征的效率-精度权衡分析

## 1. 研究题目
**常微分方程数值求解器的性能基准研究：欧拉法、龙格-库塔法与自适应步长算法在跨刚性特征系统中的效率-精度权衡分析**

## 2. 研究问题 (Research Question)
在面对具有不同刚性（Stiffness）特征的常微分方程系统时，不同阶数的数值方法（显式 vs 隐式，固定步长 vs 自适应步长）在计算成本（函数调用次数、执行时间）与全局误差之间呈现何种定量的权衡关系？是否存在一个通用的“切换阈值”来指导求解器的自动选择？

## 3. 研究动机 (Motivation)
常微分方程是模拟物理、生物、化学及工程系统的核心工具。然而，实际应用中经常遇到**刚性问题**（如化学动力学中的多尺度反应速率），其特征值的量级差异极大，导致经典的显式方法（如欧拉法、RK4）必须使用极小的步长才能维持稳定性，从而丧失效率。
虽然目前已有如 SciPy (`solve_ivp`)、MATLAB (`ode45`, `ode15s`) 和 Julia (`DifferentialEquations.jl`) 等成熟工具包，但研究者在面对复杂模型时仍难以预测哪种求解器在特定精度要求下最为高效。本研究旨在通过构建“功-精曲线”（Work-Precision Diagrams），为数值方法的选择提供实验依据。

## 4. 研究假设 (Hypothesis)
1. **非刚性系统**：自适应步长龙格-库塔法（如 Dormand-Prince 5(4)）在相同精度要求下，其计算效率将显著优于固定步长的 RK4 和欧拉法。
2. **刚性系统**：显式方法（Euler, RK4）将由于稳定性限制而产生“步长坍缩”，导致计算成本呈指数级上升甚至无法收敛；而隐式自适应步长方法（如 BDF 或 Rosenbrock）将展现出更优的稳定性权衡。
3. **效率切换点**：存在一个特定的精度阈值（Tolerance），当精度要求高于该阈值时，高阶自适应方法总是优于低阶方法。

## 5. 实验设计与方法论 (Methodology Overview)

### 5.1 实验对象（测试方程组）
本研究将选取四类具有代表性的 ODE 系统：
- **非刚性/线性系统**：简单的简谐振子方程（验证阶数精度）。
- **非刚性/混沌系统**：Lorenz 系统（测试长程演化稳定性）。
- **中度刚性/震荡系统**：Van der Pol 振子（通过参数 $\mu$ 调节刚性）。
- **极强刚性系统**：Robertson 化学动力学模型（包含三个量级差异巨大的组分，经典的刚性基准）。

### 5.2 待对比算法
1. **基础显式法**：前向欧拉法 (Euler) - 1阶，固定步长。
2. **经典高阶法**：四阶龙格-库塔法 (RK4) - 4阶，固定步长。
3. **自适应显式法**：Dormand-Prince (DP5) - 5(4)阶嵌入式，自适应步长。
4. **自适应隐式法**：二阶向后差分公式 (BDF/ODE15s 核心) 或 Rosenbrock 方法 - 针对刚性问题。

### 5.3 评价指标
- **全局误差 (Global Error)**：与解析解或极高精度参考解（如使用 128 位精度的数值解）对比。
- **函数调用次数 (Number of Function Evaluations)**：衡量计算复杂度的核心指标。
- **执行时间 (Wall-clock Time)**：衡量实际运行效率。
- **功-精图 (Work-Precision Plot)**：横轴为误差（$\log \epsilon$），纵轴为计算成本（$\log \text{Cost}$）。

## 6. 预期贡献 (Expected Contributions)
1. **多维度基准数据**：提供一套完整的实验数据，展示不同算法在刚性与非刚性系统下的性能分界点。
2. **算法选择指南**：根据系统刚性比（Stiffness Ratio）和所需的容错范围，绘制求解器建议选择矩阵。
3. **步长控制分析**：量化分析自适应步长算法在跨越系统局部刚性区域时的步长调整行为。

## 7. 范围与局限 (Scope & Limitations)
- **范围**：本研究集中于初值问题 (IVP) 的单步法和部分多步法，不涉及边界值问题 (BVP) 或随机微分方程 (SDE)。
- **局限**：计算效率受编程语言实现（如 Python 与 C++ 的差异）影响较大，因此本研究将统一在 Julia 语言环境下进行测试，以排除语言层面的开销干扰。

## 8. 相关工作定位 (Related Work Positioning)
本研究立足于以下经典与前沿工作：
- **经典理论**：Hairer, Nørsett, 和 Wanner 的《Solving Ordinary Differential Equations I & II》奠定了数值方法的理论基础。
- **现代基准**：参考了 Chris Rackauckas 等人发起的 `SciMLBenchmarks.jl` 项目，该项目对比了现代软件栈（SciPy, MATLAB, Sundials, Julia）的求解器效率。
- **改进方向**：本实验将特别引入对“刚性自动探测”机制（如 LSODA 算法中的切换机制）的评估，这是目前主流科学计算软件库的核心逻辑。
